## Round 2：修 A 組，順手揪出新問題

把 A1/A2/A3 攤開修的時候，發現三個問題其實是同一個根因：**「算」和「用」
被拆到不同 phase，圖沒把時序理順**。修法：

- **去重（dedup）搬到 Phase 2**，去重的同時順手算出 `raw_count` /
  `deduped_count` / `dedup_rate` 三個數字，當作 evidence.json 的 metadata
  一起往下傳（順便解掉問題 6）
- **命中則數計數搬到 Phase 3**，用去重後的 Evidence 算，並且明確跟「規則
  直翻」的量化訊號放一起——不是 LLM 的工作（這點下面會再展開）
- **三態約定（#8）正式定位在 Phase 4**，跟 04 對齊：Phase 3 只負責產生
  訊號＋標記「方向是否可判定」，真正決定「進票/降權進票/不進票」的是
  Phase 4 的 w 計算

修完之後重新檢查這三處改動，冒出兩個 Round 1 沒發現的新問題：

**新問題 A：w 公式從 3 因子變 4 因子，跟 03 的定義不一致**

03 的票權公式是 `w = 新鮮度 × 來源等級 × 覆蓋度`（三因子）。#3 的降權
（重複率高→降權）如果直接乘進 w，就變成第四個因子，但 03 從頭到尾只講三個。
需要回答：這個降權是併進「覆蓋度」（反正 #2 已經把覆蓋度重定義成「去重後
獨立來源數」），還是獨立算第四項？

判斷：**不能併，要獨立**。覆蓋度量的是「去重後還剩幾個獨立來源」（來源
多樣性），重複率量的是「去重前灌了多少水」（噪音/協同炒作的量）。舉例：
10 則新聞、8 則是同一篇通稿轉載、2 則是獨立報導，去重後可能剩 3 個獨立
來源（3 個獨立來源看起來覆蓋度還行），但重複率 = 1-3/10 = 0.7，這個高
重複率本身就是「可能有人在洗量」的訊號，覆蓋度算不出這件事。所以維持
獨立第四因子，但要在文件裡把這個「為什麼不能併」的理由寫清楚，不然下一個
接手的人會覺得是重複算兩次而把它刪掉。

**新問題 B：兩種降權機制會疊加，需要說明不是 bug**

#3（重複率高→降權）和 #6（拉粉話術詞庫命中→來源等級降一級）都是 quality
因子的降權，如果同一則新聞兩條都中（既是聯播稿又用了話術），w 會被壓
兩次。這是設計上允許的（兩個維度各自成立），但沒寫清楚的話，實作或答辯
時會被問「這是不是重複扣分」。需要補一句：兩者是獨立維度，允許疊加，
不是 bug。

## Round 3：修 B/C/D 組，檢查有沒有連鎖問題

依序修剩下 8 條：

  數字）
- B5 → 限制欄來源清單補全六條：#3 敘事註記、#4 背離警示、#5 LLM 解析
  失敗、#6 話術降權紀錄、#7 schema 驗證不過、#8 方向不可判定

- B7 → legend 改成完全照 04 優先序排（9→7→5→2→1→8→4→3→6），不再用分組
  排序又聲稱是優先序，兩者衝突
- C8 → 措辭改成「Phase 3 全程唯一一次**分類** call」，並在 Phase 6 旁註記
  「全程共兩次 LLM call：Phase 3 分類 + Phase 6 敘事」
- C9 → 統一用 `static/weights_backtest.json`（較具體）。**這條沒辦法在
  本檔完全收斂**——分岔的兩處寫法都在 03 裡，03 是 Ken 的檔案，需要他
  點頭才能改，本檔只能記提案，見下方「未完全收斂項目」
- C10 → 補一段名詞定義：覆蓋度（quality 因子，Phase 2/4，去重後獨立來源
  數）vs 覆蓋率（Phase 6 信心公式輸入，collector 成功執行的比例）
- D11 → 加最小樣本門檻：去重前則數 <5 時，dedup_penalty 固定為 1（不觸發
  降權），Phase 5.5 的敘事註記也跟著不觸發，避免小樣本除零或雜訊誤判

檢查這輪修改有沒有連鎖新問題：D11 的門檻值（5 則）目前是拍腦袋定的，跟
04 的「權重/門檛都要答得出出處」原則有點矛盾——但這屬於「賽前用歷史資料
校準」的範疇，不是邏輯錯誤，列進待驗證項目即可，不算新的問題輪。

三輪之後沒有再冒出新問題，視為收斂。

## 最終方案

### 完整流程圖

```
CLI (main.py) --coin/--question --dry-run
        │
        ▼
agent/orchestrator.py run_pipeline()（計時、deadline 控管）
        │
┌─ Phase 1：資料蒐集（asyncio.gather 平行執行） ────────────────────────┐
│                                                                        │
│  A. 本地層（零網路風險，最先完成）                                    │
│  csv_metrics（新）  主辦方 CSV 本地算：技術指標＋雙幣相對指標         │
│                      （相關係數/beta/相對強弱） [層1]                 │
│  calendar（新）      賽前預快取 FOMC/CPI/解鎖日靜態 JSON [層3]        │
│                                                                        │
│  B. 既有五 collector（原地升級，不重寫）                              │
│  price               即時報價 [層1] ＋ Coinbase 溢價（機構買壓代理）  │
│  onchain             依鏈路由，維持現狀 [層2]                         │
│  news                只收原始文章 [層2]，不在此計數                   │
│                       （命中則數延後至 Phase 3，去重後才算）           │
│  social               熱度情緒 [層1]；403 標記缺失並揭露               │
│  macro               F&G 百分位＋美元敏感度＋穩定幣供給 [層3]         │
│                      ＊比較題只抓一次，兩幣共用                        │
│                                                                        │
│  C. 新增 collector（繼承 BaseCollector，同一套 timeout/隔離）         │
│  derivatives（新）   funding擁擠度/OI四象限/多空比/到期結構 [層1]     │
│                      Deribit選擇權(BTC/ETH) [層2]；Polymarket [層3]   │
│  liquidity（新）     訂單簿深度/價差（微結構）[層1]                   │
│                                                                        │
│  各自獨立 timeout(75s) + try/except 隔離，失敗記 execution_log 標     │
│  skipped/error，不中斷其他 collector                                  │
└────────────────────────────────────────────────────────────────────┘
        │ list[EvidenceDraft]
        ▼
┌─ Phase 2：證據化 ──────────────────────────────────────────────────┐
│  標題相似度去重（本地字串級，不用 LLM）——先去重再往下走              │
│  同步算出 raw_count／deduped_count／dedup_rate，隨 evidence 一起傳    │
│  assign_evidence_ids() → evidence.json                              │
│  每筆新增欄位：layer（層1/2/3）、quality（新鮮度/來源等級/覆蓋度）    │
│    └ 來源等級：查 static/source_reputation.json（賽前信譽表 ＋一行   │
│                 分級理由，印進報告附錄）                               │
│    └ 覆蓋度：定義＝去重後的獨立來源數（≠下面 Phase 6 的覆蓋率，見    │
│               「名詞定義」）                                          │
└────────────────────────────────────────────────────────────────────┘
        │ list[Evidence]（含 dedup metadata）
        ▼
┌─ Phase 3：訊號化（規則優先，LLM 只碰文字） ──────────────────────────┐
│  量化訊號（規則直翻，本地計算，含 RSI、funding百分位、深度、基差、    │
│  以及去重後的「命中則數」——這是規則訊號，不經過 LLM）                │
│  文字訊號：LLM 批次分類新聞語調（全程唯一一次「分類」call；Phase 6   │
│  另有一次「敘事」call，全程共兩次）                                  │
│  拉盤話術詞庫本地匹配（"to the moon"/"100x"/公關稿模板）——命中則該   │

└────────────────────────────────────────────────────────────────────┘
        │ list[Signal]
        ▼
┌─ Phase 4：票權計算（機制①：今天誰有票、票值多少——事實，非主觀） ────┐
│  w = 新鮮度 × 來源等級 × 覆蓋度 × dedup_penalty（第四因子，獨立於     │
│      覆蓋度——覆蓋度量來源多樣性，dedup_penalty 量灌水程度，不可併， │
│      理由見「名詞定義」）公式印進報告附錄，可審計                     │
│  dedup_penalty：raw_count<5 時固定=1（樣本不足不判讀）；否則依        │
│                 dedup_rate 分級降權                                  │
│                                                                        │
│  三態約定正式落在這裡（訊號只有三態，禁止「待驗證」第四態）：         │
│    1. 進票：quality 三因子（+dedup_penalty）完整                     │
│    2. 降權進票：quality 有缺，照公式自然降權                         │
│    3. 不進票、只進限制欄：schema 驗證不過（回200但是垃圾，同樣觸發   │
│       w=0）或方向無法判定，兩者共用同一條 w=0＋execution_log 退場路徑 │
│  來源掛掉／schema 驗證不過／方向無法判定 → w=0 自動退場               │
│  （成功判準＝過 schema 驗證：筆數>0/必要欄位齊/數值合理，未過視同    │
│    error，不是「沒拋例外」就算數）                                    │
│                                                                        │
│  選配疊加（賽前算好的靜態常數，現場只做乘法，接不上就等於×1）：      │
│    機制②歷史回測權重：5年CSV算訊號歷史相關性                        │
│    機制③反變異數權重：1/歷史波動                                    │
│    → 統一存 static/weights_backtest.json（提案；03 現有兩個檔名並存, │
│      需 Ken 點頭統一，見「未完全收斂項目」）                           │
└────────────────────────────────────────────────────────────────────┘
        │ 加權訊號
        ▼
┌─ Phase 5：敏感度分析（機制⑤：結論的 stress test，非單一權重的答案）─┐
│  五組情境票制：等權／價格重／倉位重／鏈上重／宏觀重                  │
│  各組結論 = Σ(情境係數 × w × 方向)，純本地數學，毫秒級               │
│  五組同向 → 結論穩健，信心上調                                      │
│  有翻盤   → 記錄哪組翻、被哪個訊號翻 → 自動進限制欄                  │
│  輸出：結論矩陣 + 穩健度分數                                        │
└────────────────────────────────────────────────────────────────────┘
        │ 結論矩陣＋穩健度＋翻盤紀錄
        ▼
┌─ Phase 5.5：後處理檢查（純本地，零成本，接在敏感度分析之後） ────────┐
│  敘事-資金背離：新聞語調方向 vs（價格動能+funding方向），方向背離     │
│  → 產出「敘事-資金背離」警示，進限制欄                                │
│  重複率敘事註記：dedup_rate 高（且 raw_count≥5）→ 限制欄寫「新聞      │
│  熱度可能來自聯播非獨立關注」（降權已在 Phase 4 做完，這裡只是把      │
│  同一個數字轉成報告用的白話提示，不是第二次降權）                     │
└────────────────────────────────────────────────────────────────────┘
        │ 結論矩陣＋穩健度＋翻盤紀錄＋背離警示＋重複率註記
        ▼
┌─ Phase 6：推理與報告（LLM 只敘事，不算分） ──────────────────────────┐
│  信心 = f(覆蓋率, 穩健度) 查表算出，LLM 無權上調                      │
│  （覆蓋率＝collector 成功執行比例，≠ Phase 2 的「覆蓋度」，見下方定義）│
│  LLM 拿訊號＋矩陣＋證據寫敘事，逐句引用 evidence_id                  │
│  限制欄來源（六條，自動生成不靠 LLM 自覺揭露）：                      │
│    #3 重複率敘事註記／#4 敘事-資金背離／#5 LLM解析失敗降權／          │
│    #6 話術詞庫命中降權／#7 schema驗證不過退場／#8 方向無法判定退場    │
└────────────────────────────────────────────────────────────────────┘
        │
        ▼
最終報告：市場判斷 ＋ 關鍵依據 ＋ 信心與限制
```

### 名詞定義（C10 的產出）

| 名詞 | 定義 | 用在哪 |
|---|---|---|
| 覆蓋度 | 去重後的**獨立來源數**（來源多樣性） | Phase 2/4，票權 quality 三因子之一 |

   照 04 的「答得出出處」原則，賽前如果有空應該用歷史新聞量分佈校準一次，
   沒空就先用 5 當預設值上場
