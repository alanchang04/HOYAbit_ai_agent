# 產品脈絡與合規紅線

## 命題目標

本專案為「2026 雲湧智生：臺灣生成式 AI 應用黑客松競賽」HOYA BIT 命題之參賽作品。
目標：建構加密貨幣市場分析 AI Agent，從五類資料來源（價格/鏈上/新聞/社群/總經）
蒐集證據，經四步推理（含正反方辯論），產出具可回溯性的分析報告。

### 核心價值主張

- **可回溯性**：報告中每個判斷都能追溯到具體證據（evidence_id）
- **信任提煉**：原始資料經分層過濾（L1–L5），每層的剔除/降權皆有理由
- **誠實揭露**：資料缺口、推理失敗、信心來源全部透明呈現
- **15 分鐘硬限**：整體流程須在 15 分鐘內完成（含 degraded mode 退化機制）

## 合規紅線（絕對禁止）

1. **禁止投資建議**：所有輸出（報告/面板/baseline）不得出現以下語彙或等價表述：
   - buy / sell / hold / 進場價 / 停損點 / 停利 / 建議買入 / 建議賣出
   - 任何暗示特定交易行動的措辭
2. `INTACT` 一詞在本專案語境中僅代表「分析完整性」，不得誤用為投資相關含義
3. **不得捏造資料**：面板若資料缺失須誠實顯示缺口，不以假資料填充
4. **LLM 後端合規**：正式繳交版本須使用 Bedrock（`LLM_BACKEND=bedrock`）；
   Gemini 僅供開發階段使用，正式比賽不得使用
5. **隱私與安全**：不得在程式碼或輸出中硬編碼 API key，一律走 `.env` 環境變數

## 題型支援

| 題型 | 說明 |
|---|---|
| `multi_source` | 單幣種多面向分析 |
| `hypothesis_test` | 假說驗證（看多/看空理據對比） |
| `comparison` | 雙幣種比較分析（含相對指標） |

- 證據原文：保留英文引用
- 過濾詞典：英文為第一優先（來源語料為英文）
